# Artifact 3 – Implementing

## Selected Capability
Group Status – Health & Location Tracking

### Why this capability matters
The Fellowship travels through uncertain and dangerous environments. Being able to quickly view and update the health and location of each member ensures coordination and safety.

## Design Rationale

### Supporting Intent
The interface provides a centralized overview of all members, allowing quick assessment of group condition. It reduces friction by keeping interactions simple and direct.

### Reflection of Wireframe
The structure follows the wireframe from Artifact 2:
- A dashboard listing all members
- A detail section for updating a selected member
- Clear separation between overview and interaction

### What was not implemented
- No real data persistence
- No automatic GPS tracking
- No alerts or notifications
- No interaction logic (purely static)

### Assumptions / Constraints
- Users manually update their status
- The system prioritizes clarity over complexity
- Built as a static prototype without backend functionality
